# Next Feature Priorities

1. Scenario library packaging: add curated end-to-end example suites for ascent, sensor fault, terrain, target intercept, and missile-dynamics workflows with expected output notes.
2. Results comparison dashboard: provide a first-class web view for comparing multiple run directories, highlighting deltas in trajectory, controls, sensor quality, events, and summary metrics.
3. Campaign result persistence: store campaign definitions, run membership, aggregate metrics, and rerun metadata so browser campaigns can be reopened and audited.
4. Replay timeline tools: add event markers, sensor-validity bands, engagement annotations, and synchronized chart/canvas scrubbing.
5. Scenario-builder import and export: support validated import of external scenario JSON, named draft export, and conflict-safe overwrite prompts.
6. Report studio integration: mount configurable report generation in the workbench with selectable plots, artifact links, and static HTML export.
7. Navigation filter diagnostics: expose innovation traces, covariance summaries, fault flags, and estimator health scores in CLI and web reports.
8. Environment authoring: add guided terrain, wind, atmosphere, and ground-contact editors with preview plots and validation advisories.
9. Performance and scale checks: add benchmark fixtures for long runs, campaign throughput, memory growth, and large replay files.
10. Packaging and release automation: add reproducible local build scripts, versioned sample outputs, and release checks for Python and web assets.
